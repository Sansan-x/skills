# 5GC上下文严重性评估规则

## 概述

5G核心网（5GC）的架构特殊性决定了通用漏洞在5GC部署环境中可能有不同的严重性。本文档定义了从通用Go漏洞严重性到5GC上下文严重性的调整规则。

## 5GC架构安全模型

### 网络架构分层

```
┌─────────────────────────────────────────────┐
│                 外部网络 (DN)                  │  信任等级: 不可信
├─────────────────────────────────────────────┤
│              UPF 数据面                       │  信任等级: 低
├─────────────────────────────────────────────┤
│              N4 接口                          │  信任等级: 中
├─────────────────────────────────────────────┤
│         SBI 服务化接口层                       │  信任等级: 中-高
│   (AMF, SMF, UDM, AUSF, NRF, PCF, NSSF)     │
├─────────────────────────────────────────────┤
│              N2 接口 (NGAP)                   │  信任等级: 中
├─────────────────────────────────────────────┤
│              gNB (基站)                       │  信任等级: 低-中
├─────────────────────────────────────────────┤
│              UE (终端)                        │  信任等级: 不可信
└─────────────────────────────────────────────┘
```

### 攻击面分析

| 攻击面 | 可达性 | 说明 |
|--------|--------|------|
| N1 (UE-AMF) | 无线接入 | NAS消息，经gNB中转 |
| N2 (gNB-AMF) | 有线回传 | NGAP消息 |
| N3 (UE-UPF) | 数据面 | GTP-U封装的用户数据 |
| N4 (SMF-UPF) | 核心网内部 | PFCP会话管理 |
| N6 (UPF-DN) | 外部网络 | 用户上行数据出口 |
| SBI (NF间) | 核心网内部 | HTTP/2 RESTful API |
| N32 (SEPP间) | 漫游互联 | 运营商间接口 |

## 严重性调整规则

### 规则1：控制面 vs 数据面

**控制面NF（AMF, SMF, UDM, AUSF, NRF, PCF, NSSF, SEPP）**
- DoS类漏洞：通用 MEDIUM → 5GC **HIGH/CRITICAL**
  - 理由：控制面NF服务中断影响全网用户的注册、认证、会话管理
- 认证绕过：通用 HIGH → 5GC **CRITICAL**
  - 理由：绕过5G-AKA可导致非法终端接入网络
- 信息泄露：通用 MEDIUM → 5GC **HIGH**
  - 理由：可能泄露SUPI、密钥等敏感用户身份数据

**数据面NF（UPF）**
- 包处理漏洞：通用 MEDIUM → 5GC **HIGH**
  - 理由：UPF处理高吞吐量数据流，漏洞影响范围大
- 隧道操作漏洞：通用 HIGH → 5GC **CRITICAL**
  - 理由：GTP隧道劫持可导致用户流量窃取

### 规则2：外部可达 vs 内部接口

**外部可达接口（N1, N2, N3, N6, N32）**
- 所有漏洞严重性 **+1级**（LOW→MEDIUM, MEDIUM→HIGH, HIGH→CRITICAL）
- 理由：攻击者可从网络远程触发

**纯内部接口（SBI, N4）**
- 保持通用严重性评估
- 但如果漏洞可通过链式利用从外部接口到达：**+1级**

### 规则3：用户数据相关

**处理用户身份数据的组件（UDM, AUSF, UDR）**
- 信息泄露漏洞：通用级别 → 5GC **至少HIGH**
  - 理由：SUPI/SUCI、认证向量、订阅数据属于高敏感个人信息
  - GDPR/数据保护法规合规要求

**处理用户通信内容的组件（UPF N6出口）**
- 内容窃取漏洞：通用级别 → 5GC **CRITICAL**
  - 理由：通信内容拦截在大多数法域属于严重违法行为

### 规则4：高可用性要求

**全网级影响的NF（NRF, NSSF）**
- NRF：服务注册/发现的单点，DoS影响所有NF间通信
- NSSF：网络切片选择，异常可导致切片隔离失败
- 所有可用性漏洞：5GC **CRITICAL**

**签约用户级影响的NF（AMF, SMF）**
- AMF/SMF：承载具体用户会话，漏洞影响该NF服务的用户群
- DoS漏洞：5GC **HIGH**

### 规则5：合规性影响

**3GPP安全规范要求**
- TS 33.501：5G安全架构要求
- TS 33.117：安全保障方法和测试要求
- TS 33.512-33.519：各NF安全保障规范

违反以下3GPP安全要求的漏洞自动提升为 **至少HIGH**：
- 认证机制（5G-AKA, EAP-AKA'）
- 加密保护（NAS加密, SUPI隐私保护）
- 完整性保护（NAS/RRC完整性, N32接口完整性）
- 隔离要求（网络切片隔离）

## 上下文调整矩阵

| 通用严重性 | 控制面+外部接口 | 控制面+内部接口 | 数据面+外部接口 | 数据面+内部接口 |
|-----------|---------------|---------------|---------------|---------------|
| CRITICAL  | CRITICAL      | CRITICAL      | CRITICAL      | CRITICAL      |
| HIGH      | CRITICAL      | HIGH          | CRITICAL      | HIGH          |
| MEDIUM    | HIGH-CRITICAL | MEDIUM-HIGH   | HIGH          | MEDIUM        |
| LOW       | MEDIUM        | LOW-MEDIUM    | MEDIUM        | LOW           |

## 5GC攻击场景参考

### 场景1：通过NAS消息触发AMF漏洞
```
攻击链: UE → gNB(N2) → AMF
触发方式: 构造畸形NAS Registration Request消息
影响: AMF崩溃/代码执行 → 大面积用户无法注册
严重性调整: 外部可达 + 控制面 = CRITICAL
```

### 场景2：通过SBI接口伪造NF注册
```
攻击链: 已入侵的NF / 网络攻击者 → NRF SBI
触发方式: 伪造NF注册请求（若NRF缺少mTLS或证书验证不严）
影响: 伪造NF获取用户数据 / 中间人攻击
严重性调整: 控制面 + 认证绕过 = CRITICAL
```

### 场景3：UPF数据面缓冲区溢出
```
攻击链: UE → gNB(N3) → UPF
触发方式: 构造超长或畸形GTP-U数据包
影响: UPF进程崩溃 / 可能的代码执行
严重性调整: 数据面 + 外部可达 + 高吞吐量 = CRITICAL
```

### 场景4：SMF PFCP会话操纵
```
攻击链: 已入侵的UPF / 内网攻击者 → SMF(N4)
触发方式: 伪造PFCP Session Report
影响: 流量劫持、QoS策略绕过
严重性调整: 控制面 + 隧道操作 = HIGH-CRITICAL
```

### 场景5：网络切片隔离绕过
```
攻击链: 攻击者 UE → AMF → NSSF
触发方式: 构造指定S-NSSAI的注册请求，利用NSSF切片选择逻辑缺陷
影响: 接入未授权的网络切片 → 跨切片攻击
严重性调整: 隔离突破 + 外部可达 = CRITICAL
```
